# Lesson: NimBLE-Arduino 2.5.0 Breaking API Changes

**Date**: 2026-06-21
**Context**: Setting up BLE HID keyboard on ESP32-S3 with NimBLE-Arduino

## What We Learned
NimBLE-Arduino 2.5.0 introduced breaking API changes that cause compilation errors if using old API names. The library moved to more explicit BLE GAP mode naming.

## API Change Mapping

| Old API (pre-2.5.0) | NimBLE 2.5.0 API | Notes |
|---|---|---|
| `adv->setAdvertisementType(ADV_TYPE_IND)` | `adv->setConnectableMode(BLE_GAP_CONN_MODE_UND)` | More explicit naming |
| `adv->setAdvertisementType(ADV_TYPE_SCAN_IND)` | `adv->setConnectableMode(BLE_GAP_CONN_MODE_DIR)` | Directed connectable |
| `adv->setAdvertisementType(ADV_TYPE_NONCONN_IND)` | `adv->setConnectableMode(BLE_GAP_CONN_MODE_UND)` | Non-connectable |
| `adv->setScanResponse(true)` | `adv->enableScanResponse(true)` | Renamed for clarity |
| `service->start()` | **Removed** — no-op in 2.5.0 | Services auto-start |
| `BLE_HCI_ADV_TYPE_ADV_IND` | `BLE_GAP_CONN_MODE_UND` | Constant renamed |

## Compilation Errors You'll See

If using old API with NimBLE 2.5.0:
```
error: 'class NimBLEAdvertising' has no member named 'setAdvertisementType'
error: 'class NimBLEAdvertising' has no member named 'setScanResponse'
error: 'BLE_HCI_ADV_TYPE_ADV_IND' was not declared in this scope
```

## How to Check Your Version
In `platformio.ini`, check the library dependency:
```lib_deps =
    h2zero/NimBLE-Arduino@^2.5.0
```

## Prevention
- Always check the installed NimBLE version before copying code from tutorials
- Most online tutorials use pre-2.5.0 API — map to new API when using NimBLE 2.5.0+
- When you see compilation errors about missing members, check for API renames first
